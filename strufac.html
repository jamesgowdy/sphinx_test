strufac Module¶

class strufac.ParsePDB(pdbfile)[source]¶

Parse a PDB file for info needed to calculate structure factors The PDB is parsed according to the following format described here. As well as the atom and hetatm records the parser will extract:

  • The symmetry operators from the mandatory remark 290 record
  • The unit cell parameters from the cryst1 record
  • The deorthoganilsation matrix from the scale record.

A RuntimeError is raised if these are not available. We use the scale matrix to transform the cartesian world coordinates and symmetry operators to the fractional crystallographic basis.

Parameters:

pdbfile (str) – path to a PDB file
Returns:

instance of ParsePDB with the following attributes
Variables:
  • xyz_w – world coordinates for the assymetric unit as column vectors
  • xyz_c – fractional coordinates for the assymetric unit as column vectors
  • n – occupancies
  • Biso – isotropic B-factors
  • e – element symbols
  • S – the scale matrix for deorthoganlisation
  • abc – unit cell lengths (in Angstroms)
  • angles – unit cell angles (in radians)
  • Rw – list of symops rotation matricies for cartesian basis
  • tw – list of symops translation vectors for catesian basis
  • Rc – list of symops rotation matricies for crystallographic basis
  • tc – list of symops operator translation vectors for crystallographic basis
recalc(dec=4)[source]¶

If the scale matrix is changed using the calc_scale() method then we must recalulate the fractional basis coordinates and symops

Parameters:dec (int) – number of decimal places to round the Rc matricies to
Example::
pdb = ParsePDB(‘example.pdb’) pdb.S, pdb.SI = calc_scale(pdb.abc, pdb.angles) pdb.recalc()
