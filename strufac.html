class strufac.ParsePDB(pdbfile)[source]¶

Parse a PDB file for info needed to calculate structure factors The PDB is parsed according to the following format described here. As well as the ATOM and HETATM records the parser will extract:

  • The symmetry operators from the mandatory REMARK 290 record
  • The unit cell parameters from the CRYST1 record
  • The deorthoganilsation matrix from the SCALE record.

A RuntimeError is raised if these are not available.

Parameters:

pdbfile (str) – path to a PDB file
Returns:

instance of ParsePDB
Variables:
  • xw – x-coordinate (cartesian world basis)
  • yw – y-coordinate (cartesian world basis)
  • zw – z-coordinate (cartesian world basis)
  • n – atomic occupancies
  • Biso – isotropic B-factors
  • e – element symbol
  • S – deorthoganlisation (SCALE) matrix
  • abc – unit cell lengths (in Angstroms)
  • angles – unit cell angles (in radians)
  • Rw – symmetry operator rotation matricies for cartesian basis
  • tw – symmetry operator translation vector for catesian basis

Once parsed the above varibles can be accessed as attributes for example to calculate the coordinates in the crystallographic basis:

pdb = ParsePDB('example.pdb')
for xyz in zip(pdb.xw, pdb.xw, pdb.zw):
    xc, yc, zc = numpy.dot(pdb.S, xyz)
    print(xc, yc, zc)
calc_fraccoord(xyz=None)[source]¶

Transform cartesian world coordinates into the fractional crystallographic basis using the matrix multiplication:

\[\begin{split}\begin{bmatrix} x_c \\ y_c \\ z_c \end{bmatrix} = \mathbf{S} \begin{bmatrix} x_w \\ y_w \\ z_w \end{bmatrix}\end{split}\]

if no vector is provided the parsed world coordinates are transformed and a matrix of fractional coordinates with dimensions (3, N) is returned.

Parameters:xyz – Optional world coordinate vector
Returns:transformed fractional coordinates
